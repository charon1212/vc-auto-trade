DynamoDBで管理するデータ
・約定履歴(ExecutionHistory)…取引所側の情報
・発注(Order)…こちら側が出した注文情報や、その後の状態を管理する
・日価格情報…始値・終値・最大・最小情報

発行したいクエリ
・直近1時間の約定履歴をすべて取得する
・直近1日の約定履歴をすべて取得する
・直近1カ月の日価格情報をすべて取得する
・確認中()の発注情報の一覧を取得する
・買取完了()の発注情報の一覧を取得する
・売買完了()の発注情報の一覧を取得する


■PK=<ProductCode>EXECのレコード
・約定履歴を表す。
・SKはtimestampでエポック時を表す。ただし、1分=60000msの倍数で切り上げる。
・ExecutionListは次の配列を指定する。配列長は6固定で、順番はtimestampの昇順とする。
[{
  timestamp: number, // 10秒=10000msごとのタイムスタンプ。
  price: number, // 取引価格の取引量を重みにした加重平均
  sellSize: number, // 売り注文によって成立した取引量
  buySize: number, // 買い注文によって成立した取引量
  totalSize: number, // 合計取引量
}]
